<Callout title="GambiarraScript">
  The language where you don't close a block with `}` or `end` — you close it
  with `acabou_finalmente`, because coding in Brazil is exactly that: it took
  some work, but thank God it's finally over.
</Callout>

```gambiarrascript
mostra "Salve, tropa!"

bota nome = "Erik"
bota idade = 25

se_colar idade >= 18
    mostra nome + " pode entrar"
se_nao_colar
    mostra "volta daqui a pouco"
acabou_finalmente

gambiarra dobra(n)
    funciona n * 2
acabou_finalmente

pra_cada i de 1 ate 3
    mostra "dobro de " + i + " = " + dobra(i)
acabou_finalmente
```

GambiarraScript is a dynamic, interpreted language written in Go. It ships a
lexer, parser, a tree-walking interpreter and a bytecode VM — both first-class
and capable of running every example in this docs. There is also an LSP, a
formatter, a VSCode extension and a WebAssembly runtime that runs in the
browser.

> The keywords are Portuguese memes (gíria BR). They are **fixed** — there is
> no English alias for them. The code you write looks the same in any docs
> language; only the prose around it is translated.

## The Brazilian way

GambiarraScript is not a language for production (good thing) — it's an
**intentional gambiarra**: every keyword is something you'd hear at a Brazilian
party — `se_colar`? `acabou_finalmente`? `arruma...quebrou`? The idea is to
laugh a little while learning how a language works under the hood.

The interpreter source (in Go) is open and short enough to poke around — start
at `cmd/gs/main.go` to see how everything fits together.
